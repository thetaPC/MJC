Midterm: Broken VM


I received a broken VM for my midterm to fix it.
The VM would not boot. It kept giving this error:
kernel panic not syncing vfs unable to mount root fs on unknown-block 0 0 ubuntu
Enter:
Alt+3
I had to go into the advance options for Ubuntu and selected the oldest recovery boot out of the two since the newer recovery boot also would not boot.
Recovery boot gave me several options to select from before leaving the safe mode.
I selected:
dpkg
Once it was done, I left the safe mode and once it booted. I restarted the VM to check that it fixed the boot error.
The VM booted without any issues.
Another way to fix it.
The VM wouldn't boot because the user updated a bad kernel. You can simply delete the new version of kernel and revert back.
