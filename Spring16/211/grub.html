Booting Different ISOs with Grub Boot Loader and a USB


This document was modified from: http://www.pendrivelinux.com/boot-multiple-iso-from-usb-via-grub2-using-linux/
Plug in an empty flash drive since this process will delete anything on your drive onto your Linux system.
It is very important to be root user to continue.
Type: su - or sudo su or sudo su -
Type: df
This command will confirm that the flash drive is connected to your computer. My flash drive is on the last line.
Type: fdisk -l (<- Lowercase L)
This will let you what your flash drive is called. It should be sdx (x can be any letter) In my case, my drive is sdc.
Type: fdisk /dev/sdx
Don't forget to replace the x with your appropriate letter.
Type: d
This will delete any existing partitions.
Type: n
This will create a new partition.
Type: p
For a primary partition.
Type: 1
Creates the primary partition.
Press: Enter
This will use the default sector.
Press: Enter
Again using the default.
Type: a
For active.
Type: 1
This will make the first partition bootable.
Type:
t For partition type.
Type: c
For the use of Fat32 partition.
Type: w
This will write the changes and close fdisk.
Type: umount /dev/sdx1
This will unmount the flash drive.
Type: mkfs.vfat -F32 -n MULTIBOOT /dev/sdx
This will format the partition as Fat32.
Type: mkdir /mnt/USB
A new directory called USB will be created under mnt.
Type: mount /dev/sdx1 /mnt/USB
The flash drive will connect again.
Type: grub-install --force --no-floppy --boot-directory=/mnt/USB/boot /dev/sdx1
Grub will be installed on the drive.
Type: cd /mnt/USB/boot/grub
The directory will be changed.
Type: wget pendrivelinux.com/downloads/multibootlinux/grub.cfg
The Grug configuration will be downloaded. This file can be modified to add newer/different ISO versions. This file must have the appropirate ISO or the usb won't boot the ISO correctly.
The next step is very important! This could cause the whole process to be a waste of time since the usb won't boot properly.
Type: sync
Type: cd /mnt/USB
The directory will be changed.
I will be downloading Tiny Core for this example. But I will list the other available links for downloadable ISOs that are already listed in the Grub configuration file.
Type: wget "http://releases.ubuntu.com/13.04/ubuntu-13.04-desktop-i386.iso" -O ubuntu.iso
Again, the next step is very important! This could cause the whole process to be a waste of time since the usb won't boot properly.
Type: sync
Type: reboot
When your system is booting up, make sure you select Boot Options and select USB option. This should lead you to the Grub Boot Loader with a few ISO system. Don't be alarmed when you see multiple OSs. This is because the Grub configuration file has the ISOs in a loop to be read. But without the appropriate download, the boot won't work.
Linux Mint: wget "ftp://ftp.heanet.ie/pub/linuxmint.com/stable/15/linuxmint-15-cinnamon-dvd-32bit.iso" -O linuxmint.iso
Ubuntu 13.04: wget "http://releases.ubuntu.com/13.04/ubuntu-13.04-desktop-i386.iso" -O ubuntu.iso
These are the other two ISOs that I have successfully been able to boot.
This is what your Grub configuration file should look like minus the highlighted text. But you can insert a newer/different ISO on your USB by adding it in the Grub config like I have. You just need to know the correct files. I will explain this later.
