Ram based OS


My class was lucky to be given a few servers this semester. I am currently working with a SuperMicro Super Server 6015B-T+, model: L5-420. It has a bios of Phoenix. The server has an extended memory of 24G, Intel Xeon processor, and 2.50 GHz CPU speed. The server does not have any operating systems installed or any RAIDS. Therefore, the first step will be to set up a RAID.
Enter:
Alt+3
This will boot you into the 3ware Bios Boot Manager.
I have already done this, hence, the harddrive having: (test1)
Highlight the hard drive and select it by pressing Enter. Tab your way to:
Create Unit
Select it with Enter.
You can create a RAID name under the:
Array Name
Make sure that you add volume size under the:
Boot Volume Size
My server has 232 boot size.
Important! Make sure you select:
OK
to save what you have done. F8 will not save your changes. You can now reboot. While the server boots, the screen should say RAID successful if the process was done correctly.
Now we move to the next step.
Booting an OS with a USB drive.
Click here if you want to do this from a Linux command line.
I usedUniversal USB to install an OS quickly with GUI. TinyCore CorePlus was installed on the USB.
Plug the USB to the server. If you reboot to start the OS, you still will NOT boot into TinyCore. This is because the server has not configured to boot a USB first.
Go into the boot options of the bios. I have already done this step, hence the reason why my USB is already first on the list. Originally, the USB was under #8. Highlight #8 and press:
x
to disable it. Then go down to the appropriate USB and press:
x
to enable it. Now you can press:
+ (plus sign)
to move the appropriate USB to the very top of the list.
Now go to the Exit tab. Save and exit. Reboot.
After rebooting the server, the bootable USB will finally be able to do its job. I was able to boot into TinyCore. I have yet played with any other operating system.
